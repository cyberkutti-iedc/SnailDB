# Contributing to SnailDB

Thank you for considering contributing to SnailDB! This document outlines the guidelines for contributing to this project. Please take a moment to review these guidelines before getting started.

## Code of Conduct

By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md). Please report any unacceptable behavior to the project team.

## How to Contribute

### Reporting Bugs

If you find a bug, please ensure that it hasn't been reported before. You can check the existing issues on the [GitHub issues page](https://github.com/your-username/snaildb/issues). If the bug is not reported, please open a new issue with a clear and detailed description of the problem, along with steps to reproduce it.

### Requesting Features

Before requesting a new feature, check the existing issues to see if it has already been discussed. If not, you can open a new issue and describe the feature you would like to see, providing as much detail as possible.

### Pull Requests

We welcome pull requests! Here's a general guide:

1. Fork the repository to your GitHub account.
2. Create a new branch for your feature or bug fix.
3. Make your changes and ensure tests pass.
4. Write clear and concise commit messages.
5. Push your changes to your fork and submit a pull request.

We'll review your changes and provide feedback. Once everything is ready, your pull request will be merged.

## Development Setup

To set up a development environment:

1. Clone the repository: `git clone https://github.com/your-username/snaildb.git`
2. Navigate to the project directory: `cd snaildb`
3. Create a virtual environment: `python -m venv venv`
4. Activate the virtual environment:
   - On Linux/Mac: `source venv/bin/activate`
   - On Windows: `.\venv\Scripts\activate`
5. Install dependencies: `pip install -r requirements.txt`
6. Make your changes.

## Code Style

Please follow the [PEP 8](https://www.python.org/dev/peps/pep-0008/) style guide for Python code. We use an `.editorconfig` file to help maintain consistent coding styles. Ensure your editor supports EditorConfig or configure it accordingly.

Thank you for contributing!
